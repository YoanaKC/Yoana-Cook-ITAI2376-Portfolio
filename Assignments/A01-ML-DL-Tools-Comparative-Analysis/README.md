## Overview

A comprehensive comparative analysis of the major machine learning and deep learning tools and frameworks used in the field today. This assignment established a foundation for understanding when to use each tool and why — a practical framework for real-world AI development decisions.

## What Was Covered

- Compared TensorFlow, Keras, PyTorch, and Scikit-learn across key dimensions: ease of use, performance, deployment options, and community support
- Analyzed use cases where each framework excels (research vs. production, computer vision vs. NLP vs. tabular data)
- Examined hardware acceleration support (GPU/TPU) across frameworks
- Evaluated the ecosystem around each tool: visualization, debugging, model serving

## Key Findings

**TensorFlow/Keras:** Best for production deployment and scalability. TensorFlow Serving, TFLite, and TensorFlow.js extend models across server, mobile, and browser environments. Keras's high-level API reduces development time significantly.

**PyTorch:** Preferred in research due to its dynamic computation graph (define-by-run), which makes debugging and custom architecture experimentation more intuitive. Has become the dominant framework in academic publications.

**Scikit-learn:** The standard for classical ML (Random Forest, SVM, gradient boosting). Not designed for deep learning but remains essential for structured/tabular data and as a baseline comparison tool.

**Framework Selection Principle:** The right tool depends on the problem type, data format, deployment target, and team expertise — not on which framework is newest or most popular.
